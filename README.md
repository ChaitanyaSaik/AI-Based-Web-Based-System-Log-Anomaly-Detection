
# AI-Based Web Log Anomaly Detection System

## Overview
This project presents an **AI-driven system for detecting anomalies in system logs** using Machine Learning. Modern distributed systems generate millions of log messages every day. Manually analyzing these logs to identify failures, abnormal patterns, or cyber threats is extremely difficult and time-consuming.

This project automates log monitoring by applying **unsupervised machine learning techniques** to detect abnormal log patterns. The system processes raw log files, transforms textual log entries into numerical features, and identifies anomalies using the **Isolation Forest algorithm**.

A **Streamlit web application** provides an interactive interface where users can upload logs, analyze them in real time, and visualize anomalies.

---

# Problem Statement

Large-scale software systems generate huge volumes of logs that contain valuable operational information. However:

- Logs are **unstructured text data**
- Systems generate **millions of log entries daily**
- Manual monitoring is inefficient
- Critical failures may remain unnoticed

The goal of this project is to build a **machine learning-based anomaly detection system** that automatically identifies abnormal log patterns and alerts system administrators.

# Dataset

This project uses the **HDFS (Hadoop Distributed File System) log dataset**, which contains system log messages generated by a distributed computing environment.

Typical log entry example:

INFO dfs.DataNode$PacketResponder: PacketResponder 1 for block blk_38865049064139660 terminating

The dataset contains both:
- **Normal log events**
- **Abnormal log events (failures or unexpected patterns)**

# System Architecture

The system follows a **machine learning pipeline architecture** consisting of multiple stages.

Raw Logs → Preprocessing → Feature Engineering → Model Training → Anomaly Detection → Visualization Dashboard

## Architecture Workflow

1. Raw logs are collected from system sources.
2. Logs are cleaned and structured during preprocessing.
3. Log messages are converted into numerical vectors using TF-IDF.
4. Isolation Forest model is trained on the transformed data.
5. The model predicts whether each log entry is normal or anomalous.
6. Results are visualized through a Streamlit dashboard.

# Machine Learning Model

## Isolation Forest

Isolation Forest is an **unsupervised anomaly detection algorithm**.

The key idea is:

> Anomalies are rare and different from normal observations. Therefore, they can be isolated quickly in a random decision tree.

Instead of profiling normal data points, Isolation Forest isolates anomalies directly.

### Mathematical Intuition

Isolation Forest works by building multiple **random decision trees**.

The anomaly score of a data point is calculated as:

s(x, n) = 2 ^ (-E(h(x)) / c(n))

Where:

- x = data point
- n = number of samples
- E(h(x)) = expected path length of x
- c(n) = average path length of unsuccessful search in Binary Search Tree

Interpretation:

- If score ≈ 1 → anomaly
- If score ≈ 0 → normal

This allows the model to detect unusual log patterns efficiently.

# Feature Engineering

Logs are textual data and cannot be directly used by machine learning models.

Therefore we convert logs into numerical features using **TF-IDF (Term Frequency – Inverse Document Frequency)**.

TF-IDF measures how important a word is in a document relative to the entire dataset.

TF-IDF formula:

TF-IDF(t, d) = TF(t, d) × log(N / DF(t))

Where:

- t = term (word)
- d = document (log entry)
- TF = term frequency
- DF = document frequency
- N = total number of documents

This transformation converts log messages into vectors suitable for machine learning algorithms.
